## 整体架构

```
┌─────────────────────── Web GUI (SPA) ────────────────────────┐
│   设计工作台      推演大厅      复盘页      设置页              │
└────────────────────┬─────────────────────────────────────────┘
              REST + WebSocket(视角过滤后的事件流)
┌────────────────────┴──────── FastAPI Server ─────────────────┐
│                                                              │
│  设计服务: 设计Agent ── 工具链(搜索/下载/MinerU) → 场景包       │
│                                                              │
│  推演引擎(harness):                                           │
│    venue_loop × N ──── 前场轨: 会场状态机                      │
│    │   主席Agent(点名/切阶段/播报)  代表Agent × N              │
│    backroom_loop ───── 后场轨: 指令队列 → DM判定流水线          │
│    interrupt_loop ──── 危机弧线触发 / Crisis Update分发        │
│         │                                                    │
│    事件总线(emit串行化, scope过滤) ←→ 书记Agent(滚动摘要)       │
│         │                                                    │
│    SQLite(事件日志 = 存档/回放/续推)     LLM调用层(角色→模型路由) │
└──────────────────────────────────────────────────────────────┘
```

## 跨文档不变量(改动任一文档时自查)

1. **一切皆事件, 事件即存档**(03) —— 任何新机制引入的状态必须可由事件重建(07§2);
2. **可见性过滤只有一份实现**(03§4) —— Agent上下文与前端推送共用, 服务端执行;
3. **LLM不掷骰子**(06§3) —— 任何随机性都走程序RNG并记录seed;
4. **key不进事件日志、不回传前端**(08§3);
5. **人类与AI走同一个ActionProvider抽象**(07§4) —— 新交互点不得为人类单开分支逻辑;
6. **prompt只从尾部生长**(11) —— 易变内容只许出现在L4任务段; 事件渲染字节级确定; 任何"从中间改prompt"的新设计都要先过缓存命中率这一关.
