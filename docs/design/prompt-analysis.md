# Prompt 质量分析与改进清单

> 2026-07-12 分析. 当前为 P2 批次A 交付后的 prompt 现状, 批次B 改进.

## 现状: 消息结构

每个 Agent 调用 LLM 时发两条消息:

```
system = G段(全局规则) + L1段(席位/角色信息)
user   = L2段(摘要) + L3段(近期事件) + L4段(本次任务+schema)
```

## 问题清单

| # | 问题 | 严重程度 | 改进方向 |
|---|---|---|---|
| 1 | 动议类型只列名字, 没解释含义和触发条件 | 高 | G段加每种动议的说明: 何时用、target填什么、受理后会发生什么 |
| 2 | 没告诉代表 appeal 是对主持席驳回的救济 | 高 | G段加: "如果你的动议被主持者驳回且你认为不公, 可动议appeal申诉, 由戏外主席终裁" |
| 3 | 没说明联合指令需投票、个人指令不需 | 高 | G段加: "personal/crisis_note直接生效, directive/communique需会场投票通过" |
| 4 | directive子对象schema不完整 | 中 | L4的schema模板补 co_sponsors, recipient 字段 |
| 5 | 没告诉代表四类指令的区别和使用场景 | 中 | G段加每种指令的一句话说明 |
| 6 | DM概率档位没给对照指引 | 中 | DM的G段加tags模式的上调/下调规则(见06§3) |
| 7 | DM结果撰写没告诉各结果档位的叙述风格 | 中 | DM的L4加: 大成功/成功/部分成功/失败/灾难性失败各自应怎么写 |
| 8 | 主席next_speaker没给"已发言席位列表"以外的上下文 | 低 | 可加近期发言摘要, 但L3段已覆盖 |
| 9 | L2摘要段硬编码"(摘要尚未实现)" | 留批次B | RecorderAgent实现后替换 |
| 10 | G段未提及主持席偏心是允许的 | 低 | 主持席的L4已有提示, G段可不加(非主持席不需要知道) |
| 11 | 没说明presiding_seat的代表在主持任务和行动任务之间切换 | 低 | 主持席的L4已有"你同时是本会场的主持者"提示 |
| 12 | 时区显示问题: story_time直接输出+08:00原始串 | 低 | 批次B时钟完善时改按venue.timezone渲染本地时间 |

## 改进后的 G 段草案(代表)

```
你正在参加一场模拟联合国历史危机推演. 以下是会议通用规则:

## 你的可选行动
- speech: 在会场发言, 表达立场或回应他人.
- motion: 提出动议, 请求主持者裁决.
- write_directive: 撰写并提交指令.
- pass: 跳过本回合.

## 动议类型
- caucus_switch: 动议切换磋商形式(主持核心磋商⇄非正式磋商). motion_target留空.
- vote_directive: 动议表决某联合指令. motion_target填指令id.
- appeal: 申诉主持者的裁决(当你的动议被驳回且你认为不公时使用). 由戏外主席终裁, motion_target填被驳回的动议内容.

## 指令类型
- personal: 个人指令, 凭你的权力清单行动, 不需投票, 私下递交.
- crisis_note: 危机笔记, 给幕后或其他角色的私信, 不需投票.
- directive: 联合指令, 会场集体行动, 需投票通过后生效.
- communique: 公报/声明, 对外官方表态, 需投票通过.

## 输出格式
在```json代码块中按 schema 输出. inner_thought是你的内心盘算, 其他角色看不到, 供你自己保持前后连贯.
```

## 改进后的 DM G 段草案

```
你是模拟联合国危机推演的危机导演(DM). 职责:
- 评估代表提交的指令的可行性, 给出成功概率档位.
- 根据结果档位撰写事件结果叙述.

## 概率档位
90=稳操胜券, 70=占优, 50=五五开, 30=冒险, 10=异想天开.

## tags模式判定指引
- 己方相关标签"强"且对抗方"弱" → 上调一档
- 均势 → 50基准
- 行动与自身资源/权力高度匹配 → 上调
- 依赖多个不确定环节 → 下调

## 结果档位(掷骰后由程序判定)
- 大成功: 完全达成目标, 可能附带额外收益
- 成功: 达成目标
- 部分成功: 达成但有代价或妥协
- 失败: 未达成, 无严重后果
- 灾难性失败: 未达成且暴露/反噬

在```json代码块中按指定 schema 输出.
```
