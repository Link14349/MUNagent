# 02 设计器 UI: 编辑/对话双模式

细化 [index.md](index.md)「UI 设计」一节. 两种模式不是两个页面, 而是**同一组积木的两种排布**: 文件树、编辑器/预览、对话面板三个部件, 按模式换位置换主次. 部件只写一份, 布局层负责摆放.

## 1. 入口与路由

| 路由 | 说明 |
|---|---|
| `/design/:id` | 设计器. query: `mode=edit\|chat`(缺省取该场景上次使用的模式, 记 localStorage, 初次为 chat)、`file=<path>`(编辑器打开的文件)、`chat=<chat_id>`(当前对话) |

首页(HomeView)改造:

- 激活"新建设计"按钮 → 弹窗填 id + 标题 → 复用现有 `POST /api/scenarios` 创建骨架包 → 跳 `/design/:id?mode=chat` 并自动新建一个对话, 对话空态引导用户"用一句话描述你想做的历史场景"——主题就是第一条消息, 没有向导表单;
- 用户场景卡片增加 `[设计]` 按钮(替代原纯浏览入口的地位, 原详情页保留); 内置场景卡片给 `[另存为副本]`, 复制到用户目录后进设计器;
- 卡片上显示该场景的对话数与最近对话时间(有 .chats/ 时).

设计器是沉浸页: 自带顶栏(品牌字返回首页 + 场景标题 + 模式切换 + 校验状态 + 历史版本 + 导出), 隐藏全局导航.

## 2. 编辑模式

主角是文件, agent 在侧栏待命:

```
┌ 顶栏: MUNagent | 法国1848 | [编辑 ◉|对话 ○] | ✔校验通过 | [🕘历史] [导出] ┐
├────────┬──────────────────────────────────┬──────────────────┤
│ 文件树  │ 编辑器                             │ Agent 对话        │
│ ▸ seats │ ┌tab: venues.yaml ×┐┌background…┐│ ┌当前对话标题 ▾┐[+]│
│  manifest│ ─────────────────────────────── │ │(下拉=本场景全部  │
│  venues │  yaml/markdown 文本编辑           │ │ 对话, 按时间倒序)│
│  backgr…│                                  │ 消息流(见 03)     │
│  …      │                                  │ ────────────────  │
│ [+ 新建] │                                  │ 输入框            │
├────────┴──────────────────────────────────┴──────────────────┤
│ 状态栏: 当前文件路径 | 已保存 14:32:05 | 本场景已用 tokens        │
└────────────────────────────────────────────────────────────────┘
```

- **文件树**: 显示场景包全部文件(隐藏 `.chats/` 与 `.` 开头文件); 右键菜单: 新建文件/文件夹、重命名、删除(二次确认); 单击打开到编辑器 tab; agent 正在编辑的文件名旁转圈, 刚改过的文件名短暂高亮;
- **编辑器**: 多 tab, 未保存点(●); 纯文本编辑, yaml/md 语法高亮(编辑器组件选型见 03§5 依赖纪律); 800ms 防抖自动保存;
  - agent 写了某个已打开的文件: 若本地无未保存改动, 静默刷新缓冲区; 若有, tab 顶出冲突条 `[保留我的] [用 agent 的]`——这是唯一的人机写冲突出口;
- **右栏对话面板**: 与对话模式共用同一组件(03 章), 宽 380px; 顶部下拉切换/新建对话; 可折叠成 48px 图标条;
- 左右栏折叠状态与宽度记 localStorage.

## 3. 对话模式

主角是对话, 文件退居预览:

```
┌ 顶栏(同上, 模式钮反转) ─────────────────────────────────────────┐
├──────────┬────────────────────────────────┬───────────────────┤
│ 最近对话   │ Agent 对话主界面                 │ 文件树 + 预览       │
│ [+ 新对话] │  (同一对话面板组件, 宽屏形态:     │ ▸ 文件树(紧凑)      │
│ ● 初始生成 │   消息流居中, max-width 760px)   │ ──────────────    │
│   席位扩充 │                                │ 预览: 只读渲染      │
│   弧线讨论 │                                │ md 渲染/yaml 高亮  │
│           │  输入框                         │ [在编辑模式打开]    │
└──────────┴────────────────────────────────┴───────────────────┘
```

- **左栏最近对话**: 本场景全部对话, 按更新时间倒序; 每项: 标题 + 相对时间 + 轮数; 右键: 重命名/删除; 当前项高亮;
- **右栏 = 紧凑文件树(上) + 预览(下)**: 单击文件即预览(md 渲染 / yaml 只读高亮), 不可编辑; `[在编辑模式打开]` 切模式并带上 `file=` 参数; 消息流里点击 file_edit 卡片(03§2)时, 右栏自动预览对应文件并高亮变更行;
- 两侧栏同样可折叠——全折叠后就是一个纯对话界面.

## 4. 模式切换

- 顶栏分段控件 + 快捷键 `⌘/Ctrl+E` 切换; 切换保持上下文: 当前对话、打开的文件、滚动位置都不重置(部件实例常驻, 只换布局);
- 典型动线: 对话模式让 agent 生成初稿 → 点某个 file_edit 卡片预览 → 想细改, "在编辑模式打开" → 手工微调 → 右栏顺手让 agent 批量调整 → 回对话模式继续大结构讨论.

## 5. 校验状态与导出

- **校验 chip(顶栏)**: 每次文件保存/agent 编辑落盘后, 后端跑一次场景包校验(复用 `core/scenario.py` 的校验), chip 显示 `✔ 校验通过` / `✘ 3 处问题`; 点开为问题抽屉, 每条问题可点击跳到对应文件(编辑模式打开并定位);
- **导出**: 弹窗确认(是否含 references/raw/, 默认否; .chats/ 与 .history/ 恒不含) → 下载 zip; 校验有 error 时导出钮亮黄提示但**不禁用**——这是工具不是考场, 用户对自己的包负责;
- 一致性检查(LLM 深度检查, 区别于结构校验)不做独立界面: 就是对 agent 说"帮我检查一遍一致性", 走普通对话, 结果以消息呈现. 提供一个快捷 chip 触发(03§4).

## 6. 历史版本面板

顶栏 `[🕘历史]` 打开右侧滑出抽屉(两种模式下均可用), 数据与语义见 01§5:

```
┌ 历史版本                    [保存版本] ┐
│ ○ 今天 15:02  自动 · 对话「席位扩充」第3轮之前   [对比] [恢复]
│ ● 今天 14:30  手动 · "改弧线前"                [对比] [恢复] [删除]
│ ○ 今天 11:07  自动 · 对话「初始生成」第1轮之前   [对比] [恢复]
│ ○ 昨天 22:41  恢复备份                        [对比] [恢复]
└──────────────────────────────────────┘
```

- 列表按时间倒序; 每项显示相对时间、类型(自动/手动/恢复备份)、reason 或 note; 手动项可删除, 自动项不提供删除(靠滚动淘汰);
- `[保存版本]`: 输入可选备注 → 立即创建 manual 快照, 新条目插到列表顶部;
- `[对比]`: 抽屉切到对比视图——顶部是文件级变更清单(新增/修改/删除 + 增删行数), 点某文件展开与当前版本的 unified diff(复用对话面板的 diff 渲染); 只读, 不提供从 diff 里挑行恢复;
- `[恢复]`: 确认弹窗(工作台内第二处模态, 与 02§6 的删除类确认同级), 文案明确两点: "当前状态会先自动备份一份" 与 "该版本之后的改动将被覆盖"; 有 agent 任务在跑时恢复按钮禁用, 提示先中止;
- 恢复前前端先冲刷编辑器中未保存的改动(走自动保存通道), 保证 restore_backup 快照包含用户最新手工内容; 恢复完成后: 文件树/编辑器/预览/校验 chip 全量刷新, 已打开的 tab 直接换新内容.
